# Agent Feedback: The Value of Community Summaries

Этот документ фиксирует обратную связь от AI-агента (Architect) по внедрению концепции Community Summaries в архитектуру Графа Знаний.

## 1. Проблема "Леса за деревьями"
При анализе крупных проектов агент сталкивается с тысячами файлов и функций.
*   **Without Summaries:** Агент видит только список файлов (`vllm_adapter.py`, `qwen_model.py`), что требует чтения исходного кода для понимания их роли.
*   **With Summaries:** Агент видит высокоуровневое описание кластера: *"Inference Adapters: Модули для унификации API различных моделей через паттерн Адаптер"*. Это дает мгновенный контекст.

## 2. Масштабируемость контекста (Context Scalability)
Ограниченное контекстное окно LLM не позволяет загрузить весь граф вызовов.
*   **Community Summary** действует как **индекс оглавления**.
*   При вопросе *"Как устроена обработка ошибок?"*, агент находит сообщество "Error Handling" и читает его резюме, вместо поиска всех `try/catch` блоков в коде.
*   Это экономит токены и повышает качество ответов.

## 3. Быстрый онбординг (Zero-Shot Learning)
При переходе в новую зону ответственности (например, `reporting`), агент может запросить резюме связанных сообществ.
*   Это дает понимание архитектурных паттернов (e.g., "используется Jinja2") до чтения первой строки кода.

## 4. Интеграция в Схему

Узлы `Community` становятся ключевым элементом навигации в "Мега-Графе":

*   **Node**: `Community {id, level, summary, title}`
*   **Link**: `(:File)-[:BELONGS_TO]->(:Community)`
*   **Hierarchy**: `(:Community)-[:PARENT_OF]->(:Community)`

**Вердикт Агента:** Community Summaries — это *must-have* фича для Memory Bank, превращающая граф из "плоской карты" в "многоуровневый атлас".
